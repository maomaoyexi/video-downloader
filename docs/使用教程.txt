YT-DLP 多平台视频下载工具 完整使用教程
适配版本：v2.1.0 WebUI 源码运行版
支持平台：YouTube / Bilibili / Twitch / Niconico / Fantia
界面方式：浏览器WebUI（运行Python主程序后自动打开浏览器）
=====================================
一、安装与依赖说明
=====================================
源码运行版需包含以下内容：
  1. 视频下载工具v2.1.0-GUI.py     （Python主程序）
  2. video_downloader文件夹         （程序模块）
  3. yt-dlp/ffmpeg等依赖程序

安装步骤（新用户）：
  1. 安装Python 3，并确保python命令可用
  2. 保持「视频下载工具v2.1.0-GUI.py」与video_downloader文件夹在同一目录
  3. 将依赖程序放到Python主程序同一目录
  4. 运行「python 视频下载工具v2.1.0-GUI.py」启动

升级步骤（老用户）：
  1. 关闭正在运行的旧版本
  2. 替换「视频下载工具v2.1.0-GUI.py」和video_downloader文件夹
  3. 原有配置、下载记录、Cookie及依赖文件全部无需改动

依赖包已包含以下文件，无需单独下载：

基础必需依赖
  yt-dlp.exe    （核心下载引擎）
  ffmpeg.exe    （音视频合并与转码）
  ffprobe.exe   （媒体信息检测）
  deno.exe      （JavaScript运行时）
缺失任意一个会直接报错无法运行。

Fantia 功能可选依赖
  fantiadl.exe
缺失不影响 YouTube/Twitch/Niconico 使用，仅 Fantia 专区功能不可用。

所有依赖exe与Python主程序放在同一文件夹内即可。

=====================================
二、首次运行须知
=====================================
本工具仅用于个人学习研究，禁止商用、侵权传播下载内容。

运行「python 视频下载工具v2.1.0-GUI.py」启动程序。
程序会自动启动内置HTTP服务器，并调用系统默认浏览器打开WebUI界面。
首次运行自动生成五个平台保存目录、配置文件settings.ini、历史记录文件。
所有下载自动按【平台 / UP主 / 主播 / 创作者文件夹】分类整理。

重要提示：
- 使用过程中不要关闭Python控制台窗口（它是后端服务器）
- 关闭浏览器标签页不会停止下载，下载完成后30秒自动退出
- 需要彻底关闭工具时，点击界面右上角红色「✕ 退出」按钮

=====================================
三、WebUI界面导航
=====================================
界面顶部有6个功能标签页：

  [下载]    单链接/播放列表/频道下载，实时进度、速度、ETA显示
  [设置]    画质/编码/音频/代理/Cookie/硬件加速全部配置，内置预设管理
  [历史]    自动记录每次下载的时间/平台/标题/状态
  [工具箱]  批量下载、更新yt-dlp、WAV转MP3、目录管理等8个工具
  [帮助]    内置完整使用文档，无需打开外部txt文件
  [关于]    版本信息、依赖状态检测

右上角：
- 红色「✕ 退出」按钮，点击后彻底关闭工具
- 「⬆ 有更新」徽标（检测到新版本时显示），点击可一键更新

=====================================
四、基本下载操作
=====================================
1. 单视频下载
   - 进入「下载」标签页
   - 在输入框中粘贴视频链接
   - 确认平台选择正确（可自动识别）
   - 点击「开始下载」按钮
   - 实时查看进度条、下载速度、已下载大小、剩余时间

2. 播放列表/频道下载
   - 直接粘贴播放列表或频道链接
   - 程序会自动识别并列出内容
   - 支持选择全部或指定集数下载

3. 停止下载
   - 下载过程中点击「停止下载」按钮
   - 程序会先登记取消状态，再安全终止yt-dlp/ffmpeg进程树
   - 任务代际隔离可阻止旧任务的迟到进度覆盖新任务状态
   - 停止请求与子进程启动发生在同一时刻时，新进程也会被拒绝接管并立即清理
   - 已下载的部分文件保留，下次下载同一链接自动断点续传

=====================================
五、自动更新功能
=====================================
程序支持自动检查更新和一键更新：

- 启动后3秒自动静默检查更新，有新版本时右上角显示「⬆ 有更新」徽标
- 点击徽标查看新版本信息，点击「一键更新」自动下载并替换
- 统一通过 GitHub Releases 检查和分发最新版本
- 仅接受可信HTTPS来源，更新文件最大限制为512MB
- 发布信息必须提供有效SHA-256；下载后还会核对文件大小、SHA-256和Windows EXE格式
- 源码运行状态不会用EXE覆盖Python主程序；需要升级源码时请手动替换.py文件和video_downloader文件夹
- 自动替换重启仅适用于打包后的Windows EXE运行版
- 也可在「关于」页面查看当前版本号

=====================================
六、画质与输出设置（设置页面）
=====================================
进入「设置」标签页，可配置以下选项：

  分辨率上限：无限制 / 4K / 2K / 1080P / 720P / 480P / 360P
  编码模式：极致画质 / 兼容优先(H.264) / AV1优先 / VP9优先
  音频模式：最高音质 / 均衡(192k) / 最小体积(128k)
  输出格式：MP4 / MKV / WebM
  分离音频格式：m4a(原生) / MP3 / FLAC / WAV
  网络代理配置（HTTP / SOCKS5）
  FFmpeg硬件加速：CPU软编码 / N卡NVENC / Intel QSV / AMD AMF
  浏览器Cookie来源：Chrome / Edge / Firefox / Brave / Opera / cookies.txt文件
  Niconico弹幕下载开关
  文件名兼容、元数据嵌入、下载封面等开关

封面说明：
  - 「下载封面」默认开启，会写出并转换为JPG封面
  - 合并输出且格式不是WebM、未启用分离音频时，会同时尝试将封面嵌入媒体文件
  - WebM、分离音频或独立音视频下载时不嵌入封面，以避免后处理冲突

配置修改后点击「💾 保存设置」按钮生效。

=====================================
七、配置预设功能（设置页面顶部）
=====================================
预设管理集成在设置页面顶部，使用下拉选择框切换：

内置三套官方预设：
  最高画质：AV1/HDR/4K全开，收藏存档用
  通用兼容：1080P+H.264，所有设备可直接播放
  最小体积：轻量化批量存档，节省空间

自定义预设：
  1. 调整好所有参数后，点击「💾 保存」按钮
  2. 输入预设名称，确认保存
  3. 需要时从下拉框选择预设，立即一键加载所有配置
  4. 不需要的预设可点击「🗑 删除」按钮删除
  5. 预设保存在 presets.json，重启不丢失，可自行备份迁移

=====================================
八、工具箱功能
=====================================
进入「工具箱」标签页，提供8个实用工具：

批量工具：
  [1] TXT批量下载
      从urls.txt读取混合平台链接批量下载，自动识别平台
      文件格式：每行一个链接，#开头为注释，空行自动跳过
      建议文件保存为UTF-8编码
      同一时间只运行一个任务，链接按顺序处理并显示当前/总数及成功/失败统计
      点击停止后会取消当前子进程并阻止后续链接继续启动
  [2] 生成链接模板
      生成带注释的urls.txt模板文件，放在程序同目录

维护工具：
  [3] 更新 yt-dlp 核心
      一键自动更新下载核心到最新版本
  [4] 清理临时文件
      清理下载产生的临时文件(.part、.ytdl等)
  [5] 生成Cookie模板
      生成Netscape格式cookies.txt模板和配置说明

目录工具：
  [6] 打开下载目录
      在资源管理器中打开下载文件夹
  [7] 打开日志目录
      在资源管理器中打开日志文件夹

音频工具：
  [8] WAV转MP3
      - 支持选择文件夹或单个文件
      - MP3比特率可选：128/192/256/320kbps（默认320kbps CBR）
      - 可选递归子目录、转换成功后删除源WAV文件
      - 已存在MP3自动跳过，失败自动清理半成品
      - 使用ffmpeg libmp3lame编码器，输出44.1kHz立体声

=====================================
九、Cookie 登录配置（最重要）
=====================================
全平台推荐使用 cookies.txt 文件模式，配置步骤：

1. Edge/Chrome/Firefox 安装扩展：Get cookies.txt LOCALLY
   （纯本地运行，不会上传Cookie数据）
2. 打开对应平台官网，登录你的账号；多平台使用可依次登录所有平台
3. 在目标平台页面打开扩展 → 导出 Netscape 格式 cookies
4. 导出文件命名为：cookies.txt
5. 放到程序同级目录（与 视频下载工具v2.1.0-GUI.py 同目录）

也可直接在设置页面选择浏览器Cookie来源（Chrome/Edge/Firefox等），无需手动导出。

适用场景：
  YouTube 会员专属、年龄限制、受限内容
  Twitch 订阅专属、会员内容
  Niconico 720P及以上画质、会员内容
  Fantia 登录可见、付费会员帖子（Fantia仅支持cookies.txt文件模式）

详细Cookie配置说明请参考「docs/cookies问题答疑.txt」。

=====================================
十、直播录制使用方法
=====================================
YouTube直播录制：
  1. 粘贴YouTube的/live或/live/形式直播链接
  2. 程序会识别为直播任务并从直播开始处录制
  3. 录制过程中显示录制时长、已下载大小、分片或速度信息
  4. 点击「停止下载」即可结束录制
  5. 文件保存到 YouTube/上传者名/直播/ 目录

Twitch直播录制：
  1. 粘贴主播频道链接（如 https://www.twitch.tv/主播名）
     注意：必须是主播频道主页，不是具体视频/剪辑页面
  2. 程序自动检测开播状态
  3. 未开播会持续等待，开播后自动开始录制
  4. 录制过程中实时显示录制时长、已下载大小、下载速度
  5. 点击「停止下载」结束录制，或主播下播后自动结束
  6. 文件自动保存到 Twitch/主播名/直播/ 目录

Niconico直播录制：
  1. 粘贴Niconico直播链接（live.nicovideo.jp 或 live2.nicovideo.jp）
  2. 程序自动识别直播流，实时显示录制进度
  3. 同样支持随时停止、断点续录

=====================================
十一、下载历史记录
=====================================
进入「历史」标签页查看下载记录：
  - 自动记录每次下载的详细信息：时间、平台、视频标题、URL、状态
  - 彩色标签区分平台：YouTube(红)/Twitch(紫)/Niconico(蓝)/Fantia(粉)
  - 绿色表示成功、红色表示失败，一目了然
  - 最多保留500条记录，支持一键清空
  - 下载完成立即通过SSE实时更新，无需刷新页面
  - 历史数据保存在 download_history.json

=====================================
十二、断点续传说明
=====================================
断点续传默认启用，无需配置：
  - 添加了 --continue 参数
  - 网络中断或手动停止后，重新下载同一URL会自动从断点继续
  - 无需从头开始下载已完成的部分
  - 配合下载归档记录机制，已完成的视频不会重复下载
  - 临时文件格式为 .part 或 .ytdl，下载完成后自动重命名为最终文件

=====================================
十三、文件保存规则
=====================================
自动分类：程序目录 / 平台名 / UP主/主播/创作者名 / 视频文件
自动去重：各平台独立归档记录（*_archive.txt），已下载内容不会重复下载
临时文件：下载中断产生的 .part/.ytdl 文件可用于断点续传
日志记录：控制台实时输出日志，WebUI界面实时显示

=====================================
十四、退出与关闭说明
=====================================
彻底关闭工具：
  点击界面右上角红色「✕ 退出」按钮 → 确认退出
  程序会：终止正在进行的下载 → 关闭HTTP服务器 → 退出进程
  页面显示「工具已关闭」提示，可安全关闭浏览器

关闭浏览器标签页：
  - 下载进行中关闭：弹出确认提示，下载继续在后台运行
  - 无下载任务时关闭：30秒后自动退出进程
  - 多标签页打开：关闭其中一个不影响其他标签页

=====================================
十五、最佳稳定配置（推荐）
=====================================
Cookie模式：cookies.txt 文件模式（全平台通用，最稳定）
画质预设：通用兼容（日常播放最稳）/ 最高画质（收藏用）自选
输出格式：MP4
音视频模式：合并输出
编码：兼容优先（H.264，所有设备可直接播放）
代理：根据网络环境配置
运行方式：所有依赖exe与主程序放同一目录，运行「python 视频下载工具v2.1.0-GUI.py」启动
关闭方式：使用「✕ 退出」按钮彻底关闭，不要直接关控制台窗口
